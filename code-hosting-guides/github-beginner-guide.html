1

GitHub 是什么

GitHub 是一个基于 Git 的远程代码托管和协作平台。

保存代码

把本地 Git 仓库推送到 GitHub 后，代码和提交历史会在线保存，换电脑也能重新 clone。

协作开发

多人可以通过分支、Pull Request 和 code review 合作开发同一个项目。

自动化流程

GitHub Actions 可以在 push 或 PR 后自动运行测试、构建、部署等任务。

一句话理解：Git 管本地历史，GitHub 管线上的仓库和协作，gh 是操作 GitHub 的命令行工具。
2

第一次使用前先确认这些事

新手最容易卡在公开权限、认证方式和远程仓库地址上。

仓库公开性

公开仓库任何人都可能看到；练习项目可以公开，真实业务、作业隐私和客户代码默认选私有。

认证方式

HTTPS 常用 token，SSH 使用密钥。认证只是在证明你有权限访问这个远程仓库。

初始化文件

建议准备 README 和 .gitignore。README 说明项目，.gitignore 避免提交依赖、日志和 .env。

git clone https://github.com/用户名/仓库名.git
cd 仓库名
git status
git add .
git commit -m "Add first note"
git push
3

提醒和警示

GitHub 会让协作更方便，也会让错误更快扩散。

不要提交密码、token、SSH 私钥、.env、数据库连接串和客户数据。即使后来删除，历史提交里也可能留下痕迹。

公开前先检查内容

公开仓库适合开源和展示，不适合未经处理的公司项目、作业答案、隐私数据和内部配置。

谨慎重写历史

git push --force、git reset --hard 可能覆盖别人工作，团队仓库不要随手使用。

先拉取再推送

多人协作时，修改前先 git pull，推送失败先看冲突，不要盲目覆盖远程。

4

推荐学习流程

按这个顺序学习，可以先跑通最小闭环，再接触协作能力。

  1. 注册 GitHub 账号，确认邮箱可用，了解公开仓库和私有仓库的区别。
  2. 安装 Git，配置 user.name 和 user.email。
  3. 创建一个练习仓库，勾选 README，先从网页看懂仓库页面。
  4. 把仓库 clone 到本地，修改 README，完成第一次 add、commit、push。
  5. 学习 pull、branch、merge，理解多人协作为什么要用分支。
  6. 创建一个 Pull Request，体验 review、合并和关闭 issue。
  7. 最后再学习 GitHub Actions 和 gh，不要把它们和 Git 基础混在一起学。
5

术语注解

先把这些词分清，命令和网页按钮就容易理解。

Repository

仓库，一个项目的代码、文档和历史记录集合。

Remote

远程仓库地址，本地 Git 通过它知道代码要推送到哪里。

Origin

默认远程仓库别名，通常指向你 clone 或关联的 GitHub 仓库。

Commit

一次提交记录，相当于项目在某个时间点的快照。

Branch

分支，用来隔离功能开发、修复和实验。

Clone

把远程仓库完整下载到本地。

Pull

把远程最新提交拉取并合并到本地。

Push

把本地 commit 上传到远程仓库。

Pull Request

请求把一个分支的修改合并到另一个分支，常用于代码评审。

Issue

问题或任务记录，可以写 bug、需求、讨论和待办。

HTTPS / SSH

两种连接 GitHub 的协议；HTTPS 常配 token，SSH 常配密钥。

Token

访问令牌，用来替代密码授权命令行访问 GitHub。

6

常用命令速查

先掌握这些命令，就能完成大多数 GitHub 新手任务。

本地版本控制

git status
git add .
git commit -m "说明"
git log --oneline
git diff

连接远程仓库

git clone 仓库地址
git remote -v
git remote add origin 仓库地址

同步远程仓库

git pull
git push
git push -u origin main

分支协作

git branch
git checkout -b feature-name
git merge feature-name

排查问题

git remote -v
git status
git log --oneline --decorate

GitHub CLI

gh auth status
gh repo view
gh pr list
gh issue list
