1

Gitee 是什么

Gitee 是一个基于 Git 的代码托管平台，功能定位类似 GitHub。

中文界面友好

Gitee 的界面、通知和协作流程对中文用户比较直接，适合国内团队和新手练习。

保存和同步代码

本地 Git 记录提交，Gitee 保存远程副本，让你可以在不同电脑之间同步项目。

协作管理

可以用 issue 记录任务，用合并请求检查和合并分支修改。

Gitee 和 GitHub 不是 Git 的替代品。Git 是工具，Gitee/GitHub 是线上平台。
2

第一次使用前先确认这些事

先选好仓库公开性和认证方式，再开始上传代码。

公开还是私有

公开仓库适合展示和开源，私有仓库适合个人练习、公司项目和不想公开的代码。

远程仓库地址

本地项目要 push 到 Gitee，必须让 Git 知道远程地址，常见别名是 origin。

HTTPS 或 SSH

HTTPS 上手快，SSH 长期使用更顺。两者只是连接同一个仓库的两种方式。

git clone https://gitee.com/用户名/仓库名.git
cd 仓库名
git status
git add .
git commit -m "添加第一份说明"
git push
3

提醒和警示

代码托管平台会保存历史记录，所以不要把敏感内容带进提交。

不要提交密码、token、SSH 私钥、.env、数据库连接串和真实用户数据。仓库设为私有也不代表可以随便提交敏感信息。

先写 .gitignore

依赖目录、构建产物、日志、环境变量文件通常不应该提交到 Gitee。

协作前先 pull

多人修改同一个项目时，先拉取远程最新代码，减少冲突和覆盖风险。

谨慎危险命令

git reset --hard 和强推可能丢失内容。新手遇到冲突先查看状态，不要急着覆盖。

4

推荐学习流程

先完成本地到远程的最小闭环，再学习分支协作。

  1. 注册 Gitee 账号，绑定邮箱，理解公开仓库和私有仓库的区别。
  2. 安装 Git，配置 user.name 和 user.email。
  3. 创建一个练习仓库，初始化 README，复制 HTTPS 地址。
  4. 把仓库 clone 到本地，修改文件后完成 add、commit、push。
  5. 学习 git remote -v，理解本地项目为什么要关联远程仓库。
  6. 创建新分支，推送分支，发起合并请求，理解评审后再合并的流程。
  7. 熟悉 SSH key，长期使用时把认证方式从 HTTPS 逐步切到 SSH。
5

术语注解

这些词在 Gitee 页面和 Git 命令里会反复出现。

Repository

仓库，一个项目的代码、文档和提交历史集合。

Remote

远程仓库地址，告诉本地 Git 要连接哪个线上仓库。

Origin

默认远程仓库别名，通常指向 Gitee 上的项目地址。

Commit

一次提交记录，描述某次修改做了什么。

Branch

分支，用来开发功能、修复问题或隔离实验。

Clone

把 Gitee 仓库下载到本地。

Pull

把 Gitee 上的最新提交拉到本地。

Push

把本地 commit 上传到 Gitee。

PR / MR

合并请求，请求把一个分支的代码合并到目标分支。

Issue

问题、需求或任务记录。

HTTPS

用网页协议连接仓库，常用账号和访问令牌认证。

SSH

用密钥认证连接仓库，配置后日常 push/pull 更方便。

Token

访问令牌，用来替代密码授权命令行访问平台。

6

常用命令速查

先掌握这些命令，就能完成大部分 Gitee 日常使用。

本地版本控制

git status
git add .
git commit -m "说明"
git log --oneline
git diff

连接远程仓库

git clone 仓库地址
git remote -v
git remote add origin 仓库地址

同步远程仓库

git pull
git push
git push -u origin master

分支协作

git branch
git checkout -b feature-name
git merge feature-name

排查问题

git status
git remote -v
git log --oneline --decorate

SSH 密钥

ssh-keygen -t ed25519 -C "你的邮箱"
ssh -T git@gitee.com

本页用于新手学习 Gitee 基础流程。真实项目中，仓库公开性、访问令牌、SSH 密钥和合并权限都需要谨慎处理。
